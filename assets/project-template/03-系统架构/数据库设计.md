## 🎯 设计原则

### 1. 命名规范
- **表名**：小写+下划线，复数形式（users, orders）
- **字段名**：小写+下划线（user_name, created_at）
- **索引名**：idx_字段名 或 uk_字段名（唯一索引）

### 2. 字段规范
- **主键**：统一使用 BIGINT UNSIGNED AUTO_INCREMENT
- **时间字段**：统一使用 DATETIME 类型
- **状态字段**：统一使用 TINYINT 类型
- **软删除**：使用 deleted_at 字段（DATETIME，默认NULL）

### 3. 索引规范
- 主键索引：所有表必须有主键
- 外键字段：必须建立索引
- 查询频繁的字段：建立普通索引
- 唯一约束：建立唯一索引

### 4. 注释规范
- 表必须有注释
- 字段必须有注释
- 特殊字段必须详细说明

## 🔍 索引优化策略

### 何时建立索引
- 频繁查询的字段
- WHERE 条件字段
- JOIN 关联字段
- ORDER BY 排序字段

### 何时不建立索引
- 数据量很小的表
- 频繁更新的字段
- 区分度很低的字段（如性别）
