## 📦 方式一：Docker部署（推荐）

### 第1步：准备服务器
```bash
# 安装Docker和Docker Compose
curl -fsSL https://get.docker.com | bash -s docker
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 第2步：获取代码
```bash
# 克隆代码
git clone [仓库地址]
cd project

# 切换到发布分支
git checkout main
git pull origin main
```

### 第3步：配置环境
```bash
# 复制环境配置
cp .env.example .env

# 编辑配置文件
vi .env
```

### 第4步：构建镜像
```bash
# 构建应用镜像
docker-compose build

# 或使用生产配置
docker-compose -f docker-compose.prod.yml build
```

### 第5步：启动服务
```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f app
```

### 第6步：初始化数据库
```bash
# 进入应用容器
docker-compose exec app bash

# 执行数据库迁移
[迁移命令]

# 退出容器
exit
```

### 第7步：验证部署
```bash
# 检查服务是否运行
curl http://localhost:8000/health

# 检查API文档
curl http://localhost:8000/docs
```

## 🔧 方式二：手动部署

### 第1步：环境准备
```bash
# 安装系统依赖
sudo apt update
sudo apt install -y [依赖包]

# 安装应用依赖
[安装命令，如 pip install -r requirements.txt]
```

### 第2步：配置服务
```bash
# 创建应用目录
sudo mkdir -p /var/www/project
cd /var/www/project

# 获取代码
git clone [仓库地址] .

# 配置环境
cp .env.example .env
vi .env
```

### 第3步：配置进程管理
使用Systemd管理服务：

创建服务文件 `/etc/systemd/system/project.service`：
```ini
[Unit]
Description=Project Service
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/project
ExecStart=[启动命令]
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl enable project
sudo systemctl start project
sudo systemctl status project
```

### 第4步：配置Nginx
创建Nginx配置 `/etc/nginx/sites-available/project`：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/project /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 🗄️ 数据库管理

### 备份数据库
```bash
# MySQL备份
mysqldump -u user -p dbname > backup_$(date +%Y%m%d).sql

# Docker环境备份
docker-compose exec db mysqldump -u user -p dbname > backup.sql
```

### 恢复数据库
```bash
# MySQL恢复
mysql -u user -p dbname < backup.sql

# Docker环境恢复
docker-compose exec -T db mysql -u user -p dbname < backup.sql
```

### 数据库迁移
```bash
# 执行迁移脚本
[迁移命令]

# 或手动执行SQL
mysql -u user -p dbname < migrations/v1.1.sql
```

## 📊 监控和日志

### 查看应用日志
```bash
# Docker
docker-compose logs -f app

# Systemd
sudo journalctl -u project -f

# 日志文件
tail -f /var/log/project/app.log
```

### 监控服务状态
```bash
# 检查服务运行状态
systemctl status project

# 检查端口监听
netstat -tlnp | grep 8000

# 检查进程
ps aux | grep project
```

## 🚨 故障排查

### 问题1：服务无法启动
**排查步骤**：
1. 查看日志：`docker-compose logs app`
2. 检查配置：`.env` 文件是否正确
3. 检查端口：是否被占用

### 问题2：数据库连接失败
**排查步骤**：
1. 检查数据库服务是否运行
2. 检查数据库连接配置
3. 检查网络连接

### 问题3：性能问题
**排查步骤**：
1. 查看CPU和内存使用率
2. 检查数据库慢查询
3. 查看日志是否有异常
