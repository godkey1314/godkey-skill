## 🐳 Docker配置

### docker-compose.yml（开发环境示例）
```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "8000:8000"
    env_file:
      - .env
    depends_on:
      - db
      - redis
    
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: dbname
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    
  redis:
    image: redis:6.2
    ports:
      - "6379:6379"

volumes:
  mysql_data:
```

### 启动Docker环境
```bash
# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 📝 配置文件模板

### .env.example
```bash
# 数据库
DATABASE_URL=mysql://user:pass@host:3306/dbname

# Redis
REDIS_URL=redis://host:6379/0

# 应用
APP_ENV=development
APP_DEBUG=true
APP_PORT=8000

# 密钥（需要生成）
SECRET_KEY=
JWT_SECRET=

# 第三方服务
[SERVICE]_API_KEY=
```

使用时复制并重命名：
```bash
cp .env.example .env
# 然后编辑 .env 填写实际值
```

---

## ⚠️ 注意事项

### 1. 敏感信息
- ❌ 绝不提交 `.env` 到Git
- ✅ 只提交 `.env.example`
- ✅ 生产环境密钥要强随机

### 2. 环境隔离
- 开发、测试、生产使用不同的数据库
- 不要在生产环境开启DEBUG模式

### 3. 密钥管理
- 定期更换密钥
- 使用环境变量，不要硬编码
