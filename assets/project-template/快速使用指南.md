# 快速使用指南

> **5分钟了解如何使用本文档体系**

---

## 🎯 这是什么

这是一套**AI友好的项目文档模板**，专为AI辅助开发设计。

**核心特点**：
- ✅ 模板驱动，AI逐步填充
- ✅ 结构清晰，易于理解
- ✅ 上下文连续，AI可接力开发

## 🚀 如何使用（人类视角）

### 第1步：复制模板到新项目
```bash
# 复制整个文档目录
cp -r ocra-doc/ your-project/docs/
cd your-project/docs/
```

### 第2步：填写基本信息
1. 编辑 `01-产品介绍/产品概述.md`
2. 编辑 `03-系统架构/技术选型.md`
3. 编辑 `06-开发规范/` 中的规范文档

### 第3步：开始AI协作开发
1. 和AI讨论需求
2. AI填充 `04-功能设计/` 中的文档
3. AI开发代码
4. AI创建开发日志

### 第4步：持续迭代
- 需求变更 → 更新功能设计文档
- 功能完成 → AI创建开发日志
- 数据库变更 → 添加SQL文件
- Git管理版本

---

## 🤖 如何使用（AI视角）

### 每次会话开始
1. 阅读 `09-AI开发日志/README_AI使用说明.md`
2. 阅读最近3-5个开发日志
3. 确认本次任务

### 开发过程中
1. 编写/更新功能设计文档
2. 创建SQL文件（如需要）
3. 开发代码

### 会话结束时
1. **必须**创建本次开发日志
2. 记录完成内容、遇到的问题
3. 记录下次要做什么

## 🎯 重点文档说明

### 1. 功能设计文档（04-功能设计/）
**最重要**：开发前必须完善此文档

**包含内容**：
- 功能目标
- 业务流程
- 数据设计
- 接口设计
- 业务规则

**使用模板**：`04-功能设计/模块设计模板.md`

---

### 2. AI开发日志（09-AI开发日志/）
**最关键**：AI连续性的保证

**文件命名**：`[日期]_Session[编号]_[任务名].md`

**内容要求**：
- 本次完成了什么
- 创建了哪些文档/代码
- 遇到的问题和解决方案
- 下次要做什么

**使用模板**：`09-AI开发日志/日志模板.md`

---

### 3. 数据库SQL（05-数据库/sql/）
**规范要求**：
- 文件命名：`[日期]_[操作]_[表名].sql`
- 包含注释
- 可独立执行

## 📋 开发规范

### Git提交规范
```bash
# 格式
<type>(<scope>): <subject>

# 示例
feat(user): 实现用户登录功能
fix(order): 修复订单计算错误
docs: 更新API文档
```

### 文档更新规范
1. **能追加就追加**：在文档末尾添加"变更记录"
2. **新文档单独创建**：不修改旧文档，创建新文档
3. **文件名包含信息**：日期+模块+功能

1. ✅ 每次会话开始阅读日志
2. ✅ 开发前完善功能设计

## 💡 最佳实践

不要一次实现太多功能，每个Session完成一个小任务。

### 2. 文档先行
在写代码前先完善功能设计文档。

### 3. 及时记录
遇到问题和解决方案都记录在日志中。

### 4. Git管理
所有文档变更都通过Git管理版本。

## ❓ 常见问题

### Q1：文档太多了，都要填吗？
**A**：不是。核心文档是：
- `04-功能设计/` - 必须
- `09-AI开发日志/` - 必须
- 其他按需填写

### Q2：AI会忘记之前的内容怎么办？
**A**：这就是为什么要强制AI创建开发日志。下次AI通过日志恢复上下文。

### Q3：文档需要写得很详细吗？
**A**：够用就好，不要过度设计。重点是：
- 功能设计写清楚
- 开发日志写完整
- 其他简洁即可

### Q4：可以用在其他项目吗？
**A**：可以！这是通用模板，复制到任何项目都能用。
