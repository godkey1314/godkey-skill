# 代码规范

> **项目代码编写规范**

---

## 📋 通用规范

### 1. 命名规范

#### 文件命名
- 小写字母+下划线：`user_service.py`
- 或驼峰命名：`UserService.java`

#### 变量命名
- **Python**：小写+下划线 `user_name`
- **JavaScript**：驼峰命名 `userName`
- **常量**：大写+下划线 `MAX_SIZE`

#### 函数命名
- 动词开头，表达清晰的动作
- `getUserInfo()` / `createOrder()` / `updateStatus()`

#### 类命名
- 大驼峰命名：`UserService` / `OrderController`

---

### 2. 注释规范

#### 文件注释
```python
"""
文件功能描述

@author: 作者名
@date: 创建日期
"""
```

#### 函数注释
```python
def get_user_by_id(user_id: int) -> dict:
    """
    根据用户ID获取用户信息
    
    Args:
        user_id: 用户ID
        
    Returns:
        用户信息字典
        
    Raises:
        UserNotFoundError: 用户不存在
    """
    pass
```

#### 行内注释
```python
# 计算总价（含税）
total_price = price * quantity * 1.13
```

---

### 3. 代码格式

#### 缩进
- **Python**：4个空格
- **JavaScript**：2个空格
- 禁止混用空格和Tab

#### 行长度
- 每行不超过 80-120 字符
- 过长的行要换行

#### 空行
- 函数之间空2行
- 类之间空2行
- 逻辑块之间空1行

## 🐍 Python规范

### 1. 遵循PEP 8
使用工具检查：
```bash
flake8 your_code.py
black your_code.py
```

### 2. 类型提示
```python
def process_data(data: list[dict]) -> bool:
    """使用类型提示"""
    pass
```

### 3. 导入顺序
```python
# 标准库
import os
import sys

# 第三方库
import requests
from fastapi import FastAPI

# 本地模块
from app.models import User
```

## 💻 JavaScript规范

### 1. 使用ESLint
配置文件：`.eslintrc.js`

### 2. 使用const/let
```javascript
// ✅ 推荐
const userName = 'John';
let userAge = 25;

// ❌ 不推荐
var userName = 'John';
```

### 3. 箭头函数
```javascript
// ✅ 推荐
const add = (a, b) => a + b;

// ❌ 不推荐（简单函数）
const add = function(a, b) {
    return a + b;
};
```

## 🗄️ SQL规范

### 1. 关键字大写
```sql
SELECT id, name
FROM users
WHERE status = 1;
```

### 2. 表名和字段名小写
```sql
-- ✅ 推荐
SELECT user_name FROM users;

-- ❌ 不推荐
SELECT UserName FROM Users;
```

### 3. 使用参数化查询
```python
# ✅ 推荐
cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))

# ❌ 不推荐（SQL注入风险）
cursor.execute(f"SELECT * FROM users WHERE id = {user_id}")
```

## 📝 Git提交规范

### Commit Message格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type类型
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

### 示例
```bash
feat(user): 实现用户登录功能

- 添加登录接口
- 实现JWT认证
- 添加单元测试

Closes #123
```

## ⚠️ 禁止事项

### 1. 禁止硬编码
```python
# ❌ 不要这样
API_KEY = "sk-xxxxxxxx"

# ✅ 应该这样
API_KEY = os.getenv("API_KEY")
```

### 2. 禁止大段注释代码
```python
# ❌ 不要留着无用代码
# def old_function():
#     pass
#     ...

# ✅ 删除或使用Git管理
```

### 3. 禁止随意打印
```python
# ❌ 不要直接print
print("Debug info")

# ✅ 使用日志
logger.debug("Debug info")
```
