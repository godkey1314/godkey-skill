# 项目文档中心

> **通用项目文档模板 - 适用于AI辅助开发**

---

## 📚 文档说明

本文档体系专为**AI辅助开发**设计，包含完整的项目文档模板。

**适用场景**：
- 新项目启动时，复制本模板到项目目录
- AI根据模板逐步填充内容
- 通过Git管理文档版本

---

## 📂 目录结构

```
doc/
├── 01-产品介绍/        产品层面文档（对外）
├── 02-使用指南/        用户使用手册
├── 03-系统架构/        技术架构设计
├── 04-功能设计/        功能详细设计（核心）
├── 05-数据库/          数据库SQL文件
├── 06-开发规范/        开发规范文档
├── 07-部署运维/        部署和运维文档
├── 08-资源文件/        图片、API导出等
├── 09-AI开发日志/      AI工作记录（重要）
└── 10-开发计划/        开发计划、重构方案、修复方案
```

---

## 🚀 快速开始

### 第1步：复制模板
```bash
cp -r doc/ your-project/docs/
```

### 第2步：AI开始工作
1. AI阅读 `09-AI开发日志/README_AI使用说明.md`
2. AI根据需求填充各模块文档
3. AI每次对话创建日志文件

### 第3步：Git版本管理
```bash
git add docs/
git commit -m "docs: 更新XXX文档"
```

## 📖 各目录说明

### 01-产品介绍
**内容**：产品概述、快速开始、核心特性  
**用途**：给外部人员了解项目  
**维护频率**：项目初期建立，后期少量更新

### 02-使用指南
**内容**：各模块使用说明（用户手册）  
**用途**：教用户怎么使用系统  
**维护频率**：功能开发完成后补充

### 03-系统架构
**内容**：架构设计、技术选型、数据库设计  
**用途**：技术人员了解系统架构  
**维护频率**：项目初期建立，架构变更时更新

### 04-功能设计
**内容**：各模块详细功能设计文档（核心！）  
**用途**：开发人员参考设计文档进行开发  
**维护频率**：开发前编写，开发中不断完善

### 05-数据库
**内容**：数据库建表SQL、迁移SQL  
**用途**：数据库初始化和版本管理  
**维护频率**：每次数据库变更时添加SQL文件

### 06-开发规范
**内容**：代码规范、Git规范、命名规范  
**用途**：团队开发保持一致性  
**维护频率**：项目初期建立，基本不变

### 07-部署运维
**内容**：环境配置、部署手册  
**用途**：部署和运维参考  
**维护频率**：部署前编写，环境变更时更新

### 08-资源文件
**内容**：Swagger导出、截图、原型图等  
**用途**：存放各种资源文件  
**维护频率**：按需添加

### 09-AI开发日志（重要！）
**内容**：AI每次对话的工作记录
**用途**：AI通过日志恢复上下文，保持开发连续性
**维护频率**：每次对话必须创建日志

### 10-开发计划
**内容**：开发计划、重构方案、修复方案、架构调整方案
**用途**：开发前的设计文档，确保方向正确再动手
**维护频率**：每次开发/重构/修复前创建
**组织方式**：按日期分目录，`YYYY-MM-DD`（单计划）或 `YYYY-MM-DD_NN`（多计划）

## 🤖 AI使用说明

### AI每次对话开始时
1. 阅读 `09-AI开发日志/README_AI使用说明.md`
2. 阅读最近3-5个日志文件，了解项目状态
3. 确认本次任务

### AI对话过程中
1. 根据需求填充/更新 `04-功能设计/` 中的文档
2. 如有SQL变更，在 `05-数据库/sql/` 中添加SQL文件
3. 开发代码

### AI对话结束时
1. 在 `09-AI开发日志/日志/` 中创建本次日志
2. 记录完成内容、创建的文档、遇到的问题

## 📋 文档编写规范

### 1. Markdown格式
- 使用标准Markdown语法
- 善用标题、列表、表格、代码块

### 2. 简洁清晰
- 一句话说清楚核心内容
- 避免废话和重复
- 重点内容用加粗或列表突出

### 3. 结构化
- 使用固定的文档结构
- 按模板填充内容
- 保持格式统一

### 4. 增量更新
- 文档末尾追加"变更记录"
- Git commit记录变更历史
- 不删除旧内容，只追加新内容

## 🎯 核心原则

1. **模板驱动**：所有文档都有模板，AI按模板填充
2. **简单实用**：不追求完美，够用就好
3. **AI友好**：文档结构清晰，AI容易理解
4. **Git管理**：文档版本通过Git管理
5. **持续更新**：开发过程中不断完善文档
