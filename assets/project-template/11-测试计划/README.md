# 11-测试计划

> 按日期组织的测试计划文档。每次需要编写测试（>3 用例或涉及集成/端到端测试）时，在此创建对应的测试计划文档。

---

## 目录结构

```
11-测试计划/
├── README.md
├── 测试计划模板.md            # 测试计划模板，创建测试计划时参考
├── YYYY-MM-DD/              # 当天只有一个计划
│   └── 测试计划_功能名.md
├── YYYY-MM-DD_01/           # 当天多个计划 → 日期_序号
│   └── 测试计划_功能名.md
├── YYYY-MM-DD_02/
│   └── 测试计划_功能名.md
```

## 命名规则

| 场景 | 目录名 | 示例 |
|------|--------|------|
| 当天只有 1 个计划 | `YYYY-MM-DD` | `2026-05-17` |
| 当天有多个计划 | `YYYY-MM-DD_序号`（2 位数字） | `2026-05-17_01`、`2026-05-17_02` |

## 文件命名

测试计划文件命名格式：`测试计划_功能名.md`。例如：

- `测试计划_用户登录模块.md`
- `测试计划_支付回调接口.md`

## 典型使用场景

- 新功能开发完成后：编写该功能的测试计划
- Bug 修复后：编写回归测试计划
- 集成/端到端测试前：编写测试计划
- 小改动（≤3 用例、纯函数）可跳过，直接在日志中简述测试策略

## AI 使用说明

1. 需要制定测试计划时，参考 `测试计划模板.md` 创建文档
2. 按当前日期创建目录，落盘后进入测试执行阶段
3. 测试完成后，在 `09-AI开发日志` 中记录，测试报告与测试计划放在同一目录
