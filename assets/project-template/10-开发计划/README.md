# 10-开发计划

> 按日期组织的开发计划文档。每次需要制定开发计划、重构方案、修复方案时，在此创建对应的计划文档。

---

## 目录结构

```
10-开发计划/
├── YYYY-MM-DD/              # 当天只有一个计划
│   └── 方案名称.md
├── YYYY-MM-DD_01/           # 当天多个计划 → 日期_序号
│   └── 方案名称.md
├── YYYY-MM-DD_02/
│   └── 方案名称.md
└── README.md
```

## 命名规则

| 场景 | 目录名 | 示例 |
|------|--------|------|
| 当天只有 1 个计划 | `YYYY-MM-DD` | `2026-05-17` |
| 当天有多个计划 | `YYYY-MM-DD_序号`（2 位数字） | `2026-05-17_01`、`2026-05-17_02` |

## 文件内容

每个计划目录下的 `.md` 文件命名采用**描述性中文名称**，直接反映计划内容。例如：

- `数据质量监控体系重构方案.md`
- `用户模块性能优化方案.md`
- `支付接口安全加固方案.md`

## 典型使用场景

- 新功能开发前：编写设计方案
- 重构前：编写重构方案
- 代码审查后：编写修复方案
- 架构调整：编写调整方案

## AI 使用说明

1. 需要制定开发计划时，按当前日期创建目录
2. 参考项目已有文档（03-系统架构、04-功能设计）编写方案
3. 方案完成后，在 `09-AI开发日志` 中记录本次计划
