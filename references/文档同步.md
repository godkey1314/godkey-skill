# 文档同步

适用场景：所有任务完成后执行（代码落盘 = 触发）。

---

## 同步规则（按改动类型）

| 改动类型 | 文档动作 |
|---------|---------|
| 小修复 / 单文件 | 只记日志 |
| 新功能 / 重构 / 大修复 | 记日志 + 补充或创建设计文档 |
| 数据结构变更 | 记日志 + 更新数据设计文档 |
| 架构调整 | 记日志 + 更新架构设计文档 |

**日志总是要记（只要代码落盘）。**

---

## AI 开发日志规范（09-AI开发日志）

### 存放位置

项目文档体系下的 `09-AI开发日志/日志/` 目录。

### 命名

`YYYYMMDD_SessionNNN_任务名.md`（编号接续已有日志）

### 内容要求

必须包含：做了什么、变更前快照、涉及文件、问题与决策、下次计划、遗留风险。

### 规则

- 同一连续任务不重复开新日志
- 追问、返工、补充继续更新原日志
- 只有切换到新任务时才创建新日志

---

## 开发计划（10-开发计划）

### 何时创建

- 新功能开发（涉及文件 > 3 个）
- 重构（改动范围跨模块）
- 架构调整
- 专项修复方案
- 用户明确要求"先出方案"

### 何时不创建

- 单文件小修复、文案调整、配置修改、格式修正

### 目录命名

```
10-开发计划/
├── YYYY-MM-DD/              # 当天一个计划
├── YYYY-MM-DD_01/           # 当天多个 → 日期_序号
└── YYYY-MM-DD_02/
```

---

## 功能设计文档（04-功能设计）

### 何时更新

**功能开发完成、代码合并后**，对照实际代码同步文档。不在开发过程中频繁更新——过程中的变更记在开发日志。

### 更新内容

API 路径/入参/出参、数据库表结构、业务流程图、关键设计决策。

---

## 文档目录结构

多模块项目（有子仓库/子模块）在根目录创建 `xxxx_doc/`，单项目在项目目录下创建 `doc/`：

```
project/
├── src/
├── doc/ 或 xxxx_doc/
│   ├── 01-产品介绍/
│   ├── 02-使用指南/
│   ├── 03-系统架构/
│   ├── 04-功能设计/
│   ├── 05-数据库/
│   ├── 06-开发规范/
│   ├── 07-部署运维/
│   ├── 08-资源文件/
│   ├── 09-AI开发日志/
│   └── 10-开发计划/
└── CLAUDE.md
```

使用此 Skill 内置的 `assets/project-template/` 初始化。

---

## 禁止行为

- 禁止代码落盘后不记日志
- 禁止文档与代码不一致（代码已改文档还写旧的）
- 禁止无脑建文档（改一行代码就建一个开发计划）
- 禁止只写不改的僵尸文档
- 禁止功能设计文档在开发过程中频繁修改（应记在开发日志）
