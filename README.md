# GodKey Skill

> 一个强制执行结构化编程方法的 Claude Code Skill —— 不跳到结论、不走捷径、不留垃圾代码。

---

## 这是什么

GodKey Skill 是一套**通用编程方法论**，编码为 Claude Code 的 Skill 格式。它的目的不是给你一堆规则条目，而是**替换 AI 的默认行为模式**：

```
原来的 AI：快速找到相关代码 → 最小改动 → 完成
加载后：  建立判断标准 → 理解完整关系 → 确定改动边界 → 精确改动 → 自检循环 → 文档同步
```

---

## 核心能力

| 模块 | 解决什么问题 |
|------|------------|
| **阅读纪律** | AI 看 6 个文件就说"全看完了"——强制追到底，输出完整文件清单 |
| **层级思维** | AI 凭感觉决定代码放哪——决策树判断每行代码的层级归属 |
| **编码原则** | AI 爱写补丁代码——十大通用编码铁律（跨语言、跨框架） |
| **迭代精炼** | AI 自检走形式——改完→对照规范→修复→再对照，循环直到零违规 |
| **文档同步** | AI 不更新文档——代码落盘后根据场景自动补日志/计划/功能设计 |
| **项目模板** | 新项目没有文档体系——内置完整文档模板，一键初始化 |

## 安装

```bash
# 全局安装（所有项目可用）
git clone https://github.com/godkey1314/godkey-skill.git ~/.claude/skills/godkey-skill
```

安装后 Claude Code 会自动识别。在你进行代码开发、审查、重构时自动触发，也可以手动触发：

> 使用 godkey-skill，帮我审查代码

---

## 适用场景

- **代码开发**：新功能、改 bug、加模块
- **代码审查**：PR Review、工作区审查
- **代码重构**：跨模块重构、架构调整
- **项目初始化**：自动搭建文档体系

不适用于：单行修改、纯问答、格式微调、文案修改。

## 设计理念

不是把编码规范写成规则清单塞给 AI（AI 会无视），而是**把决策过程编码进 Skill**：

- 不让 AI "参考一下规则" → 规定 AI "按这个流程走"
- 不让 AI "自己判断读够了" → 要求 AI "输出文件清单证明读完了"
- 不让 AI "改完就完了" → 强制 AI "对照规范逐条自检，循环修复"
