# GodKey Skill

> 通用结构化编程方法论 —— 不跳到结论、不走捷径、不留垃圾代码。

---

## 这是什么

GodKey Skill 是一个 Claude Code Skill，它替换 AI 的默认行为模式：

```
原来的 AI：快速找到相关代码 → 最小改动 → 完成
加载后：  审计文档体系 → 读规范识框架 → 设计方案 → 理解完整关系 → 定层 → 精确改动 → 自检循环 → 文档同步
```

它不是给你一堆规则条目让你参考，而是**锁定流程、聚焦注意力**——在关键节点强制 AI 停下来、输出结果，不输出就不能继续。

---

## 核心设计

| 组件 | 角色 |
|------|------|
| **SKILL.md** | 编排器：前置关卡（审计+分级处理） + 7 关 + 阻塞铁律 + 场景调度 |
| **开发流程** | 前置关卡 → 读规范识框架 → 设计方案 → 追依赖定层 → 执行自检 → 文档同步清单 |
| **代码审查** | 5 维度原则驱动式审查（非固定清单） |
| **测试流程** | 测试计划 → 确定范围 → 编写 → 运行验证 → 补漏 → 报告落盘 |
| **文档同步** | 蓝图驱动：设计文档先于代码，关卡 6 逐条打勾强制清单 |
| **项目模板** | 完整 11 目录文档体系模板，新项目一键初始化 |

## 安装

```bash
git clone https://github.com/godkey1314/godkey-skill.git ~/.claude/skills/godkey-skill
```

安装后 Claude Code 自动识别。在代码开发、审查、重构时自动触发，也可手动触发：

> 使用 godkey-skill，帮我审查代码

---

## 适用场景

- **代码开发**：新功能、改 Bug、加模块
- **代码审查**：PR Review、工作区审查
- **代码重构**：跨模块重构、架构调整
- **项目初始化**：自动搭建文档体系

不适用于：单行修改、纯问答、格式微调、文案修改。

## 设计理念

**Skill 不是知识库，是调度器。** 它不教 AI 编码规范——规范在项目自己的 `doc/` 里。Skill 只做一件事：在正确的时间，把 AI 的注意力强制指向正确的位置。

- 不让 AI "参考一下规则" → 规定 AI "从项目规范提取约束，逐条对照自检"
- 不让 AI "自己判断读够了" → 要求 AI "输出文件清单证明读完了"
- 不让 AI "改完就完了" → 强制 AI "改完自检、同步蓝图、循环修复、文档同步清单逐条打勾"
- 不让 AI "随便测一下就完" → 要求 AI "先写测试计划，再测，测完落盘报告"
- 不把文档当附属品 → **设计文档是蓝图，代码照着蓝图施工。代码可以落后，文档必须最新。**
